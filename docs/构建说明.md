# Windows 与 macOS 构建说明

项目版本只在 `src/yucedu_converter/__init__.py` 中维护，构建脚本会自动读取并校验版本。

## Windows

要求：Python 3.11 或更高版本、PowerShell。

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -e ".[build]"
.\scriptsuild_windows.ps1
.\scripts\package_release.ps1
```

输出：

```text
dist/YUCEdu双向转换器/
release/yucedu-converter-v2.1.0-windows-x64.zip
release/yucedu-converter-v2.1.0-windows-x64.zip.sha256.txt
```

Windows 构建脚本会依次执行隐私检查、自动测试、PyInstaller 构建、正式 EXE 启动检查和任务栏图标检查。

## macOS

要求：macOS、Python 3.11 或更高版本、Xcode Command Line Tools。

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -e ".[macos]"
bash scripts/build_macos.sh
bash scripts/package_macos.sh
```

输出根据当前 Mac 架构生成：

```text
dist/YUCEdu双向转换器.app
release/yucedu-converter-v2.1.0-macos-arm64.dmg
release/yucedu-converter-v2.1.0-macos-arm64.dmg.sha256.txt
```

Intel Mac 的资产名为 `macos-x64`。macOS 构建脚本会执行单元测试、Tk 检查、App 启动检查、Info.plist 检查、架构检查和 `codesign` 完整性检查。

## GitHub Actions 自动发布

`.github/workflows/release.yml` 使用三个独立环境构建：

- `windows-latest`：Windows x64。
- `macos-15`：Apple Silicon arm64。
- `macos-15-intel`：Intel x64。

推送与项目版本一致的标签后，例如：

```text
v2.1.0
```

工作流会验证版本、构建三个平台资产、核对 SHA256，并自动创建 GitHub Release。

`workflow_dispatch` 可用于正式打标签前的构建验收；手动触发只生成 Actions Artifact，不创建 Release。

## Apple 签名说明

当前 GitHub 构建采用 ad-hoc 签名，可验证 App 内部代码完整性。接入 Apple Developer ID 证书和公证凭据后，可在发布流程中继续增加 hardened runtime、notarytool 和 stapler。
