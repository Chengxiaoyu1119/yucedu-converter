# 多平台路线图

## 第一阶段：Windows 参考实现

- 公开 Windows 2.0.1 源码。
- 建立自动测试、构建和 Release。
- 将当前 Python 实现作为跨平台正确性基准。

## 第二阶段：格式规范与 CLI

- 增加 `inspect`、`encrypt`、`decrypt`、`verify` 命令。
- 建立合成测试向量和固定 SHA256。
- 把格式配置、兼容尾部和算法版本显式化。

## 第三阶段：共享核心

推荐使用 Rust 实现 `yucedu-core`：

- 流式加密与解密。
- 媒体类型识别。
- 进度与取消。
- 临时文件和原子提交。
- 配置档案与兼容测试。
- C ABI / FFI 供桌面和移动界面调用。

Python 版本继续作为参考实现，Rust 输出必须与固定测试向量逐字节一致。

## 第四阶段：桌面平台

### macOS

- Intel x86_64。
- Apple Silicon arm64。
- Universal 2 应用。
- ZIP 或 DMG 发布。

### Linux

- x86_64 与 arm64。
- CLI、AppImage 和桌面 GUI。

## 第五阶段：移动平台

### Android

- 文件导入、批量任务、后台进度、导出和内置播放。

### iOS / iPadOS

- Files 导入、转换、导出和内置播放。

## 长期产品模块

```text
yucedu-core          跨平台转换核心
yucedu-cli           命令行与自动化
yucedu-desktop       Windows / macOS / Linux
yucedu-mobile        Android / iOS
yucedu-player        流式读取与播放
yucedu-compatibility 固定测试向量和播放器兼容矩阵
```
