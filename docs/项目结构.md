# 项目结构与路径管理

仓库只保存源码、测试、文档和构建配置；生成文件与本地样本放在 Git 管理范围外。

## 精简目录

```text
yucedu-converter/
├─ .github/                    # Issues、PR 与 Actions
├─ docs/                       # 用户和开发文档
│  ├─ images/                  # README 与界面图片
│  └─ 研究/                    # 上游来源与兼容研究
├─ packaging/windows/         # Windows 打包配置
├─ scripts/                   # 构建、校验与验收脚本
├─ src/yucedu_converter/      # Python 程序源码
│  └─ resources/              # 转换资源与程序图标
├─ tests/                      # 自动测试
├─ pyproject.toml              # Python 项目配置
├─ README.md                   # 项目首页
└─ LICENSE                     # MIT 许可证
```

## 路径规则

1. 文档路径最多保留两层；内部实施计划不进入公开仓库。
2. Python 源码使用标准 `src` 布局，避免模块导入和打包路径混乱。
3. Windows 打包配置统一放在 `packaging/windows`，执行脚本统一放在 `scripts`。
4. `build`、`dist`、`release` 都是生成目录，由 `.gitignore` 排除。
5. 播放器原文件、真实媒体和回归样本保存在仓库外。
6. 文档与代码只使用相对路径、环境变量或占位符，不记录个人绝对路径。
