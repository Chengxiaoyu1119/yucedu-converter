# 项目结构、技术栈与路径规范

仓库只保存可公开、可复现的源码、文档、测试和构建配置。本机样本、参考播放器、转换结果与生成文件不进入 Git。

## 整体结构

```text
yucedu-converter/
├─ .github/
│  ├─ ISSUE_TEMPLATE/               # Issue 表单
│  ├─ workflows/
│  │  ├─ ci.yml                     # Windows 与 macOS 持续集成
│  │  └─ release.yml                # 三个平台自动构建与发布
│  ├─ CONTRIBUTING.md               # 开发规范
│  └─ SECURITY.md                   # 安全问题报告方式
├─ docs/                            # 用户、构建、发布和研究文档
│  ├─ images/
│  └─ 研究/
├─ packaging/
│  ├─ windows/                      # Windows PyInstaller 配置
│  └─ macos/                        # macOS App PyInstaller 配置
├─ scripts/                         # 版本、构建、打包和检查脚本
├─ src/yucedu_converter/            # 正式 Python 程序包
│  └─ resources/                    # 转换资源与程序图标
├─ tests/                           # unittest 自动测试
├─ local/                           # 本机私有资料，Git 忽略
├─ .editorconfig
├─ .gitattributes
├─ .gitignore
├─ LICENSE
├─ README.md
└─ pyproject.toml
```

根目录继续只保留项目入口和工具必须识别的配置文件。

## 技术栈

| 技术 | 作用 |
|---|---|
| Python 3.11+ | 转换核心、界面和平台适配 |
| Tkinter / ttk | Windows 与 macOS 桌面界面 |
| `cryptography` | AES 等密码学运算 |
| `threading` + `queue` | 后台转换与界面进度通信 |
| PyInstaller | Windows EXE 目录与 macOS App 打包 |
| PowerShell | Windows 构建与发布 |
| Bash、codesign、hdiutil | macOS 构建、完整性签名与 DMG |
| `unittest` | 无额外依赖的自动测试 |
| GitHub Actions | 双平台 CI 与 Release |

项目没有数据库、服务器后端、Electron 或网页前端。

## 主要源码

| 路径 | 作用 |
|---|---|
| `src/yucedu_converter/__init__.py` | 程序名称、唯一版本号和当前配置标识 |
| `src/yucedu_converter/__main__.py` | 支持模块启动 |
| `src/yucedu_converter/main.py` | 版本输出、冒烟启动和 GUI 入口 |
| `src/yucedu_converter/gui.py` | 主窗口、任务列表、批量处理和平台文案 |
| `src/yucedu_converter/converter.py` | 文件识别、流式转换、输出和校验 |
| `src/yucedu_converter/player.py` | Windows、macOS、Linux 播放器与文件管理器调用 |
| `src/yucedu_converter/settings.py` | 各平台设置、日志和资源路径 |
| `src/yucedu_converter/theme.py` | 跨平台字体、颜色和控件样式 |
| `src/yucedu_converter/resources/` | 查表资源、兼容尾部与图标 |

`gui.py` 与 `converter.py` 当前继续保持原有结构；本版本只增加必要平台分支。

## 构建与发布

### Windows

- `packaging/windows/yucedu-converter.spec`：Windows PyInstaller 规则。
- `packaging/windows/version_info.template.txt`：自动生成 EXE 版本资源的模板。
- `scripts/build_windows.ps1`：测试、构建和正式 EXE 验收。
- `scripts/package_release.ps1`：整理 ZIP 与 SHA256。

### macOS

- `packaging/macos/yucedu-converter.spec`：App Bundle、图标、Bundle ID 和最低系统版本。
- `scripts/build_macos.sh`：测试、构建、启动、Info.plist、架构和签名检查。
- `scripts/package_macos.sh`：生成 arm64 或 x64 DMG 与 SHA256。

### 通用

- `scripts/version_tool.py`：读取唯一版本号、校验 Git 标签、生成 Windows 版本资源。
- `.github/workflows/ci.yml`：Windows 3.11/3.13 与 macOS arm64 检查。
- `.github/workflows/release.yml`：Windows x64、macOS arm64、macOS x64 构建并创建 Release。

## 程序数据流程

```text
用户选择模式、输出目录和文件
            ↓
GUI 创建任务并交给后台线程
            ↓
converter 检查输入、资源和输出路径
            ↓
按数据块转换并发送进度
            ↓
临时文件完成校验后替换为正式输出
            ↓
player 调用当前系统播放器或打开所在目录
```

## 路径规范

1. 根目录只放入口和工具要求的配置。
2. 用户与开发说明放入 `docs/`；GitHub 协作文件放入 `.github/`。
3. Python 源码放入 `src/yucedu_converter/`，资源放入其 `resources/`。
4. 平台打包配置分别放入 `packaging/windows/` 和 `packaging/macos/`。
5. 可执行维护脚本统一放入 `scripts/`。
6. 文档、源码和配置只使用相对路径、环境变量或示例占位符。
7. 本机样本和原播放器放入 `local/`，真实媒体与账号资料不提交 Git。
8. `build/`、`dist/`、`release/`、缓存、日志和转换结果属于生成内容。
9. 版本只修改 `src/yucedu_converter/__init__.py` 的 `APP_VERSION`。
10. 文件版本通过 Git 标签和 `docs/更新日志.md` 管理。
